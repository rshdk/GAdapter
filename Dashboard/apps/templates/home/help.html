Support & Documentation

Learn how to configure the panel, build clients, and manage tasks.

1. Build Client

Before building, you need to set up a client ID which is used to identify clients when connecting:

Client Configuration 
1. Open the settings page
2. Enter a unique client ID (e.g., "client123")
3. Add Panel domain with Port [https://example.com:443]
4. Add check-in interval (e.g., 60 seconds)
5. Choose the client type (exe, Python, powershell) depending on your need
6. Click Build Client

When new clients connect, the tasks are fetched and executed automatically.

2. Create a New Task

To create a new task for client devices:

Task Creation 
1. Click "New Task" button
2. Enter the Client ID
3. Choose task type (PowerShell, EXE, or DLL)
4. Enter command or upload file
5. Click "Create Task"

Note: For DLL tasks, specify the entry point in method name if native dll or mention method and class name if C# dll.

3. Manage Auto Tasks

Auto Tasks are permanent templates that deploy automatically to checking-in clients.

Auto Task System 
• Creation: Tasks created here get a unique ID. They are securely queued for execution.
• Deployment: When a client checks in, the server sees which Active Tasks the client has NOT executed yet and assigns them. 
• Unique Targeting: Each client runs a specific Auto Task exactly once. New clients instantly run all Active tasks. Old clients will run newly created tasks next time they check in.
• PAUSE/RESUME: You can Pause a task to prevent assignments. If you Resume it later, any client that checked in while it was paused will finally receive it on their next ping.

4. Monitor Task Status

Completed Task executed successfully
Pending Task queued but not executed
Failed Task execution failed

5. View Task Results

Click on any task view button to see detailed results in the response modal. It shows:

  • Task ID and client information
  • Original command/file details
  • Full execution output
  • Error messages (if any)
